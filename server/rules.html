Правила Табличка

Несколько бесит аргумент 'А вы давали код тестировать своим друзьям?'. Давайте кооперироваться)) (если против - можете мне написать).

У меня в общаге на малинке поднят сайт (этот). Каждые ~30 минут будут запускаться тестирующие скрипты. Результаты будут заноситься в табличку.

Скрипты работают следующим образом:

  1. Скачивается репозиторий (потом возможно добавлю отслеживание изменений)
  2. Происходит сборка репозитроия
  3. Если репозиторий собрался, запускаются тесты из этого репозитория
  4. (пока ещё нет) Если пройдены тесты этого репозитория, запускаются тесты всех остальных репозиториев.
  5. Отчеты сохраняются и заносятся в табличку

!ВАЖНО! Это НЕ автоматическая тестирующая система. Будет очень плохо, если писать тесты будет только один человек. Поэтому, если хотите, чтобы ваше решение тестировалось, то следуйте следующим правилам:

  1. Тесты должны быть написаны с использованием библиотеки doctest
  2. Файл с тестами должен лежать в папке с задачкой и иметь название test.cpp или tests.cpp
  3. Тесты должны собираться в файл с именем, содержащим слово test в названии
  4. Файл с тестами НЕ ДОЛЖЕН БЫТЬ ПУСТЫМ ИЛИ БЫТЬ ЗАГЛУШКОЙ (Буду следить за их длинной автоматически и долгими холодными вечерами отсматривать их галзками. Потом будет бан)
